# ①課題名
ブックマークアプリ（CRUD機能・ログイン認証・管理者権限付き）

## ②課題内容（どんな作品か）
- PHPとMySQLを使ったブックマーク管理アプリです。
- ユーザー認証機能を実装し、ログインしたユーザーのみが一部機能を利用できるように修正
- ユーザーが書籍名、URL、コメントを入力してデータベースに保存できます。
- 登録したブックマークを一覧表示し、検索機能で絞り込むことができます。
- データの編集機能を実装しています（ログインユーザー全員が利用可能）
- データの削除機能は管理者のみが利用可能
- セッション管理を追加

## ③アプリのデプロイURL
https://pathcraft.sakura.ne.jp/php04_kadai/select.php

## ④アプリのログイン用IDまたはPassword（ある場合）
データベースの`gs_user_table`に登録されているユーザーIDとパスワードを使用
## ⑤工夫した点・こだわった点
- セッション管理により、ログインしていないユーザーは編集・削除ページにアクセスできないようにしました。
- 管理者フラグにより、削除機能は管理者のみが利用できるように制限
- ログイン状態に応じて、ヘッダーにログイン/ログアウトボタンを動的に表示しています。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- セッション管理とログイン認証の実装が難しかったです。特にセッションIDの再生成タイミングを理解するのに時間がかかりました。
- 管理者権限の実装で、権限チェックを適切な場所に配置する必要がありました。
- カテゴリ分けやタグ機能を追加して、ブックマークをより整理しやすくしたいです。
- パスワードのハッシュ化を実装して、セキュリティをさらに強化したいです。
- ユーザー登録機能を追加して、新規ユーザーが自分でアカウントを作成できるようにしたいです。
